\begin{abstract}
Many methods across statistics, optimization, and finance return not a single
solution but a \emph{path} as one scalar parameter is swept: the LASSO along its
penalty, the support-vector machine along its cost, and Markowitz's Critical Line
Algorithm, predating them all, along risk aversion; explicit model predictive
control, whose parameter is a vector state, is the multi-parameter generalisation of
the same scheme. We argue, and prove on the cleanest pair, that the scalar-parameter
methods are one device: with the covariance taken to be a Gram matrix, the constrained
mean--variance frontier and the constrained LASSO path are the \emph{same}
piecewise-linear curve, with the same breakpoints. The identity is exact, and, beyond the
unconstrained correspondence and the gross-exposure case of \citet{brodie2009}, it
survives arbitrary linear equality and inequality constraints. Its proof is short, a
completion of squares read through the affine-segment structure the family shares; the
content is that it makes a long-standing analogy precise and general, not that it is
technically deep. The
accompanying open-source package \texttt{cvxcla} makes the unification operational: it
traces every instance, efficient frontier or regularisation path, through one engine
that reaches its quadratic form only as an operator, so factor models and kernels drop
in unchanged. Around these we organise the family by its few distinguishing features,
recount how the device, invented in the 1950s, was rediscovered by statistics around
2000, and collect the open questions the unified view makes precise.
\end{abstract}

\section{Introduction}

This is a paper about one algorithmic idea that four fields invented separately and
never fully recognised as shared. The algorithm follows the optimiser of a parametrised
convex quadratic as a single scalar moves; the solution traces a
piecewise-linear curve, and an active set is all the state needed to trace it
exactly. In statistics it is least angle regression and the LASSO homotopy
\citep{tibshirani1996, osborne2000, efron2004}; in machine learning the regularization
path of the support vector machine \citep{hastie2004svm} and the elastic net and
generalized lasso \citep{zou2005, tibshirani2011}; in control the explicit linear
quadratic regulator computed by multi-parametric quadratic programming
\citep{bemporad2002, tondel2003}; and in finance, decades earlier, Markowitz's
Critical Line Algorithm for the efficient frontier \citep{markowitz1956,
markowitz1959}.

We make three claims, and we lead with them rather than with the survey. First
(Section~\ref{sec:identity}), the two most dissimilar-looking members, a portfolio
optimizer and a sparse regression, are under one substitution the same curve, with the
same breakpoints, and the identity persists under arbitrary linear
constraints. We state this as a theorem and prove it, but the proof is elementary, a
completion of squares together with the affine-segment structure of
Section~\ref{sec:scheme}; its value is that it turns a familiar analogy into an exact
and general statement, and that the constraints, which statistics usually handles only
approximately, pass through for free. Second
(Section~\ref{sec:taxonomy}), once the identity is in hand, the entire family is
organised by a short list of distinguishing features, and the apparent diversity of
the methods collapses onto that list. Third (Sections~\ref{sec:tour}--\ref{sec:comp}),
the unification is not only conceptual but \emph{operational}: the open-source package
\texttt{cvxcla} traces every instance through one engine that accesses its quadratic
form only as an operator, so structured forms (factor models, kernels) drop in and
carry the sixty-year-old algorithm to modern scale.

The synthesis is, in places, a synthesis of known parts, and we are explicit about
which (Section~\ref{sec:history} and throughout). To our knowledge no previous work has
unified the Critical Line Algorithm with LARS/LASSO, the support-vector-machine path,
the generalized lasso, and explicit model predictive control as instances of a single
parametric active-set homotopy, nor proved the exact mean--variance$\,=\,$LASSO path
identity under arbitrary linear equality and inequality constraints; the constrained
LASSO path itself, under such constraints, is due to \citet{gaines2018}. What is new
here is the exact general-constraint identity and the organisation it makes possible;
what is old, and worth retelling, is that statistics spent the 2000s rediscovering a
device finance and optimization had in the 1950s.

\section{The common scheme}
\label{sec:scheme}

Fix a convex quadratic whose linear part carries a scalar $\lambda$,
minimised over a polyhedron:
\begin{equation}
\label{eq:pqp}
\begin{aligned}
x(\lambda) = \arg\min_{x \in \R^n}\ &\tfrac12\, x\trans H x - \lambda\, d\trans x \\
\text{s.t.}\quad &A x = b,\ \ G x \le h,\ \ \lb \le x \le \ub .
\end{aligned}
\end{equation}
A polyhedral \emph{penalty} such as $\lambda\|x\|_1$ fits the same template through its
subgradient, as Section~\ref{sec:identity} makes explicit. This is a one-parameter
parametric quadratic program; its solution map has the structure that is the whole
story.

\paragraph{The standing assumption} We require only that $H\succeq0$ and that the
\emph{reduced} Hessian $H_{\F\F}$ on the free block be positive definite at every
active set the path visits. This is weaker than the textbook $H\succ0$, and
deliberately so: it is exactly what the regression instance needs. There
$H=X\trans X$ is singular whenever $p>n$, the very regime the LASSO is built for, yet
the free block $X_\F\trans X_\F$ is positive definite as soon as the active columns are
linearly independent, which holds in general position (the uniqueness remark after
Theorem~\ref{thm:identity}). The finance instance has $H=\Sig\succ0$ outright, so the
condition is automatic there. Under it the bordered solve below is well-posed, which is
all the homotopy ever touches; a globally positive-definite $H$ is never needed.

\begin{lemma}[Affine segments and the bordered solve]
\label{lem:affine}
On any maximal interval of $\lambda$ on which the active set is constant, the optimiser
of \eqref{eq:pqp} is affine,
\begin{equation}
\label{eq:affine}
x(\lambda) = \alpha + \lambda\,\beta ,
\end{equation}
and $(\alpha,\beta)$ come from a single bordered linear solve \eqref{eq:border} whose
matrix is constant on the interval. An active inequality row enters that solve
identically to an equality row.
\end{lemma}

\begin{proof}
Fix $\lambda$ in the interior of the interval and name the active set: the equalities
$A$ (always active); the active inequality rows $\Sset\subseteq\{1,\dots,p\}$, those
held at $g_i\trans x = h_i$, the remainder carrying a zero multiplier by complementary
slackness; and the blocked variables $\B$ pinned at a bound, with free set
$\F=\B^{c}$. Stack the active linear constraints as
$C=\big[\begin{smallmatrix}A\\ G_{\Sset}\end{smallmatrix}\big]\in
\R^{(m+|\Sset|)\times n}$ with right-hand side $r=(b;\,h_{\Sset})$. Because the program
\eqref{eq:pqp} is convex ($H\succeq0$) with linear constraints, the KKT conditions are
necessary and sufficient. Stationarity is
\[
Hx - \lambda d + C\trans\zeta + \rho = 0,
\]
with $\zeta\in\R^{m+|\Sset|}$ the equality and active-inequality multipliers and $\rho$
the bound multipliers, supported on $\B$. Read the $\F$ rows, where $\rho_\F=0$, and
split $Hx$ into free and blocked columns: $(Hx)_\F = H_{\F\F}\,x_\F + H_{\F\B}\,x_\B$,
with $x_\B$ fixed at its bound values. Primal feasibility of the active rows is
$C_\F x_\F + C_\B x_\B = r$. Collecting the unknowns $(x_\F,\zeta)$,
\begin{equation}
\label{eq:border}
\begin{bmatrix} H_{\F\F} & C_\F\trans \\[2pt] C_\F & 0 \end{bmatrix}
\begin{bmatrix} x_\F \\[2pt] \zeta \end{bmatrix}
=
\begin{bmatrix} \lambda\, d_\F - H_{\F\B}\,x_\B \\[2pt] r - C_\B\,x_\B \end{bmatrix}.
\end{equation}
The coefficient matrix is the saddle-point matrix of the active constraints, and it is
nonsingular. Indeed $H_{\F\F}\succ0$ by the standing assumption, and under the
linear-independence constraint qualification $C_\F$ has full
row rank; if $(u,v)$ lies in the kernel then $H_{\F\F}u + C_\F\trans v = 0$ and
$C_\F u = 0$, so left-multiplying the first equation by $u\trans$ gives
$u\trans H_{\F\F}u + (C_\F u)\trans v = u\trans H_{\F\F}u = 0$, which forces $u=0$
since $H_{\F\F}\succ0$, whereupon $C_\F\trans v = 0$ with $C_\F$ of full row rank forces
$v=0$. Now the matrix in \eqref{eq:border} depends only on $H$ and the active normals,
so it is constant on the interval, while the right-hand side is affine in $\lambda$
(the only $\lambda$ term is $\lambda d_\F$); hence $(x_\F(\lambda),\zeta(\lambda))$ is
affine in $\lambda$, and appending the $\lambda$-independent blocked entries $x_\B$
gives \eqref{eq:affine} with $\beta_\B = 0$. Each homotopy step is therefore one solve
of \eqref{eq:border}: a factorisation of $H_{\F\F}$ bordered by the $m+|\Sset|$ active
normals through its Schur complement. Finally, $G_{\Sset}$ enters \eqref{eq:border}
only as additional rows of $C$, indistinguishable from the equality rows $A$, and the
box bounds are carried by the partition $(\F,\B)$ and the fixed $x_\B$, not folded into
$C$. A polyhedral penalty $\lambda\|x\|_1$ is the same statement with $d$ replaced, on
each segment, by the constant subgradient sign pattern $-s_\Aset$ of the support; that
is the only change needed to read \eqref{eq:border} as the LASSO segment solve.
\end{proof}

The intervals partition the $\lambda$-axis at \emph{breakpoints} where the active set
must change; \citet{rosset2007} gave the clean sufficient condition (a quadratic loss
with a polyhedral penalty or constraint), and in control the same fact is the
piecewise-affinity of the mpQP solution over \emph{critical regions}
\citep{bemporad2002}; \citet{roll2008} likewise traces single-parameter
piecewise-linear solution paths in that setting, explicitly generalising the
LARS/LASSO construction. Continuity of $x(\lambda)$ across breakpoints (the free block is
uniquely determined on each segment, and adjacent segments agree at the breakpoint)
lets a \emph{homotopy} recover the whole path: from a vertex with known
active set, follow \eqref{eq:affine} to the first event, update the active set by that
one event, repeat. Each step is one solve against the active block of $H$ bordered by
the active normals. Event selection, ``the smallest positive step to the next
breakpoint,'' is the simplex method's ratio test, and it inherits the simplex
method's degeneracy hazard and its remedy: a lowest-index (Bland) tie-break that makes
the walk deterministic. We call \eqref{eq:pqp}, \eqref{eq:affine}, and this homotopy
the \emph{common scheme}.

\paragraph{$H$ is an operator, not a matrix}
It is worth stating at the outset what the homotopy actually asks of $H$, because it is
much less than ``a stored $n\times n$ matrix.'' Inspecting the step of
Lemma~\ref{lem:affine}, the quadratic form enters only through three linear actions:
a matrix--vector product $v\mapsto Hv$, a solve $Hv = r$ restricted to the active
block, and the cross product coupling active and inactive coordinates. Anything that
supplies those three actions can play the role of $H$; the matrix is never needed
explicitly. This is not a fussy implementation point but the source of the scheme's
reach. A factor risk model $\Sig = \operatorname{diag}(\delta) + U\Lambda U\trans$
with $k\ll n$ factors, or a kernel Gram matrix $H = \kappa(X,X)$, supplies the three
actions in $O(nk)$ or through the kernel trick \emph{without ever forming the
$n\times n$ object}, the factor case through the Woodbury identity. Reading $H$ as an
operator rather than a matrix is therefore what keeps the common scheme, and the
identity of Section~\ref{sec:identity}, generic across dense covariances, factor
models, and kernels alike; we return to it in Section~\ref{sec:comp}.

\section{The exact identity}
\label{sec:identity}

We now show that the two most dissimilar members of the family are, under one
substitution, the same instance of the common scheme, and that the identity is robust
to general constraints.

\paragraph{The two programs}
For weights $w\in\R^n$, expected returns $\mu$, and covariance $\Sig\succ 0$, the
gross-exposure-constrained mean--variance program is
\begin{equation}
\label{eq:mark}
\begin{aligned}
\text{M}_c:\quad \min_{w}\ &\tfrac12\, w\trans \Sig\, w - \mu\trans w \\
\text{s.t.}\quad &\|w\|_1 \le c,\ \ A w = b,\ \ G w \le h .
\end{aligned}
\end{equation}
For a response $y\in\R^m$ and design $X\in\R^{m\times n}$, the constrained LASSO is
\begin{equation}
\label{eq:lasso}
\begin{aligned}
\text{L}_\lambda:\quad \min_{\beta}\ &\tfrac12\,\|y - X\beta\|_2^2 + \lambda\,\|\beta\|_1 \\
\text{s.t.}\quad &A\beta = b,\ \ G\beta \le h .
\end{aligned}
\end{equation}
The two programs keep their fields' conventional symbols, $w$ for portfolio weights and
$\beta$ for regression coefficients; under the substitution of Theorem~\ref{thm:identity}
these are one and the same vector, and below we identify them freely.

\begin{theorem}[CLA--LASSO identity under general constraints]
\label{thm:identity}
Suppose $\Sig = X\trans X$ and $\mu = X\trans y$. Then for every $\lambda\ge 0$ there is
a $c(\lambda)=\|\beta(\lambda)\|_1$, nonincreasing in $\lambda$, such that
$\mathrm{M}_{c(\lambda)}$ and $\mathrm{L}_\lambda$ have the same minimiser. The solution
path of $\mathrm{M}_c$ as $c$ varies and the regularization path of
$\mathrm{L}_\lambda$ as $\lambda$ varies are therefore the \emph{same} piecewise-linear
curve in $\R^n$, traced by the same active-set homotopy and sharing the same breakpoints
as points of $\R^n$. The correspondence of the \emph{parameters},
$\lambda\mapsto c(\lambda)$, matches the two breakpoint sequences one-to-one whenever it
is strictly decreasing; Proposition~\ref{prop:reg} pins down exactly when it is, and the
single way it can fail.
\end{theorem}

\begin{proof}
Completing the square,
\[
\tfrac12\|y-Xw\|_2^2 = \tfrac12\, w\trans (X\trans X)\, w - (X\trans y)\trans w
+ \tfrac12\|y\|_2^2 .
\]
With $\Sig=X\trans X$ and $\mu=X\trans y$ the
objective of $\mathrm{M}_c$ equals $\tfrac12\|y-Xw\|_2^2$ up to the additive constant
$\tfrac12\|y\|_2^2$, which does not affect the minimiser. Hence, under the same linear
constraints $Aw=b,\ Gw\le h$, $\mathrm{M}_c$ is exactly the constrained least-squares
problem $\min_w \tfrac12\|y-Xw\|_2^2$ subject to $\|w\|_1\le c$ and those constraints.
Its Lagrangian relaxation of the single constraint $\|w\|_1\le c$ is
$\min_w \tfrac12\|y-Xw\|_2^2 + \lambda\|w\|_1$ subject to $Aw=b,\ Gw\le h$, i.e.\
$\mathrm{L}_\lambda$. Because the objective is convex and $\|\cdot\|_1$ is convex,
strong duality holds for the norm-ball constraint, and the constrained and penalised
forms have identical solution sets as $c$ and $\lambda$ range over $[0,\infty)$, with
$c(\lambda)=\|\beta(\lambda)\|_1$ the active constraint level at the dual value
$\lambda$; $c(\lambda)$ is nonincreasing because larger penalties shrink $\|\beta\|_1$.
Both objectives are quadratic and both feasible sets are polyhedral, so by
Lemma~\ref{lem:affine} each path is piecewise linear; since the two programs share a
minimiser at every parameter value, the two paths are the same set of points, with the
same corners, and tracing either by the homotopy of Section~\ref{sec:scheme} visits the
same vertices in the same order. The parameter map $\lambda\mapsto c(\lambda)$ is
nonincreasing; it matches the breakpoint values one-to-one exactly when it is strictly
decreasing, which is the content of Proposition~\ref{prop:reg}.
\end{proof}

\begin{remark}
The unconstrained ($A,G$ absent) case is the textbook LARS/LASSO correspondence; the
single gross-exposure case with a budget is \citet{brodie2009}. The content of
Theorem~\ref{thm:identity} is that arbitrary linear equalities and inequalities pass
through unchanged, and this is now a consequence of Lemma~\ref{lem:affine} rather than
a claim: an active inequality row appears in the bordered solve \eqref{eq:border} as a
row of $C=[\,A;\,G_{\Sset}\,]$, indistinguishable from an equality, so the segment, the
breakpoints, and the per-step cost are exactly those of the equality-only problem.
The general-purpose solvers most practitioners reach for, coordinate descent and
proximal methods, return the path only on a $\lambda$-grid, and standard LARS omits the
constraints; that a quadratic loss under polyhedral constraints still admits an exact
piecewise-linear path is itself classical \citep{rosset2007}, and the constrained LASSO
path under arbitrary linear equalities and inequalities is traced directly, by an
active-set homotopy on the same KKT system, by \citet{gaines2018}. What the identity
adds is not that path but its reading: it \emph{is} the constrained mean--variance
frontier, returned by the very loop that follows the efficient frontier, with no
separate machinery and the constraints entering the per-step solve indistinguishably
from equalities.
\end{remark}

The parameter map $c(\lambda)$ of Theorem~\ref{thm:identity} can be pinned down
exactly, which both sharpens the theorem and isolates the only way the parameter
bijection can fail.

\begin{proposition}[Regularity of the $c$--$\lambda$ correspondence]
\label{prop:reg}
Let $\beta(\lambda)$ be the constrained LASSO path and
$c(\lambda)=\|\beta(\lambda)\|_1$ as in Theorem~\ref{thm:identity}. Then $c$ is
continuous and piecewise linear, affine on each segment with
\[
\begin{gathered}
\beta_\Aset(\lambda) = \alpha_\Aset - \lambda\,\beta_{\mathrm{slope},\Aset},
\qquad s_\Aset = \operatorname{sign}(\beta_\Aset), \\
c'(\lambda) = -\,s_\Aset\trans \beta_{\mathrm{slope},\Aset},
\end{gathered}
\]
the sum taken over the current support $\Aset$. In the unconstrained case
$\beta_{\mathrm{slope},\Aset} = (X_\Aset\trans X_\Aset)^{-1} s_\Aset$, so
\[
c'(\lambda) = -\, s_\Aset\trans (X_\Aset\trans X_\Aset)^{-1} s_\Aset \;<\; 0
\]
on every segment with nonempty support, and $c$ is a strictly decreasing,
piecewise-linear bijection from $(0,\lambda_{\max}]$ onto $[0,\|\beta(0)\|_1)$. In the
constrained case $\beta_{\mathrm{slope},\Aset}$ is the free block of the solution of
\eqref{eq:border} with right-hand side $(s_\Aset;\,0)$, and
$s_\Aset\trans\beta_{\mathrm{slope},\Aset}\ge 0$; the strict inequality, and hence the
bijection, holds whenever the reduced Hessian on the active constraint tangent space is
positive definite and $s_\Aset$ is not annihilated by it.
\end{proposition}

\begin{proof}
On a segment the support $\Aset$ and the signs $s_\Aset$ are fixed, so
\[
\|\beta(\lambda)\|_1 = \sum_{j\in\Aset} s_j\,\beta_j(\lambda)
= s_\Aset\trans(\alpha_\Aset - \lambda\,\beta_{\mathrm{slope},\Aset}),
\]
affine in $\lambda$ with the stated slope; continuity across breakpoints follows from continuity
of $\beta(\lambda)$. Unconstrained, the segment stationarity
$X_\Aset\trans X_\Aset\,\beta_\Aset = X_\Aset\trans y - \lambda\,s_\Aset$ gives
$\beta_{\mathrm{slope},\Aset} = (X_\Aset\trans X_\Aset)^{-1}s_\Aset$, and
$s_\Aset\trans(X_\Aset\trans X_\Aset)^{-1}s_\Aset > 0$ because
$X_\Aset\trans X_\Aset\succ0$ and $s_\Aset\neq0$ on a nonempty support; a strictly
negative derivative on every segment makes $c$ a strictly decreasing bijection onto its
range. In the constrained case, eliminating $\zeta$ from \eqref{eq:border} by its Schur
complement expresses the free block of the response to a right-hand side $(g;0)$ as
$Z(Z\trans H_{\F\F} Z)^{-1}Z\trans g$, where the columns of $Z$ are a basis for the null
space of $C_\F$ (the active constraint tangent space). With $g=s_\Aset$ this gives
$\beta_{\mathrm{slope},\Aset}$ on the support, so
\[
s_\Aset\trans\beta_{\mathrm{slope},\Aset}
= \big\|(Z\trans H_{\F\F}Z)^{-1/2}Z\trans s_\Aset\big\|_2^2 \ge 0,
\]
with equality only when $Z\trans s_\Aset = 0$, i.e.\ when $s_\Aset$ lies in the span of
the active normals.
Otherwise it is strictly positive and the bijection follows as before.
\end{proof}

\begin{remark}[Uniqueness]
When the columns of $X$ are in general position the unconstrained LASSO solution is
unique at every $\lambda$ \citep{tibshirani2013}. Under general position together with
the linear-independence constraint qualification at each visited active set, the
constrained path of Theorem~\ref{thm:identity} is likewise unique and its breakpoint
sequence finite, so the homotopy of Section~\ref{sec:scheme} is well defined and the
identity is between two genuinely single-valued curves.
\end{remark}

\paragraph{Bound versus penalty}
The identity has a structural reading. In $\mathrm{M}_c$ the sparsity is governed by
an \emph{explicit} constraint, the gross-exposure ball; in $\mathrm{L}_\lambda$ it is
\emph{induced} by a penalty through its subgradient. The next proposition records
exactly when the two descriptions coincide and when the explicit one becomes vacuous.

\begin{proposition}[The gross-exposure hinge]
\label{prop:hinge}
For a long-only, fully invested portfolio ($w\ge 0$, $\mathbf{1}\trans w = 1$) the
gross-exposure constraint is vacuous, since $\|w\|_1 = \mathbf{1}\trans w = 1$ for all
feasible $w$; the constraint $\|w\|_1\le c$ binds only for $c\ge 1$, where it is
inactive. The long-only mean--variance problem is therefore a \emph{non-negative}
LASSO with the equality $\mathbf{1}\trans\beta=1$, and it traces a frontier rather than
a sparsity path. Short positions admitted under a binding cap $\|w\|_1\le c$ with
$c>1$ make the constraint active, and then $\mathrm{M}_c$ and $\mathrm{L}_\lambda$
coincide as in Theorem~\ref{thm:identity}.
\end{proposition}

\begin{proof}
Immediate from $\|w\|_1=\sum_i |w_i| = \sum_i w_i = \mathbf{1}\trans w$ when $w\ge0$,
so the budget fixes $\|w\|_1=1$ and $\|w\|_1\le c$ cannot bind for $c\ge1$; for $c<1$
the long-only budget is infeasible. With short positions $\|w\|_1>\mathbf{1}\trans w$
in general, and the cap binds for $c$ below the unconstrained $\|w\|_1$.
\end{proof}

regularisation path traced by the homotopy and the gross-exposure-constrained
mean--variance solutions, solved \emph{independently} as quadratic programs, are the
same curve.

\paragraph{The portfolio face, concretely}
A small example fixes the portfolio reading. We simulate $T=50$ daily returns for
$N=20$ assets from a $K=5$ factor model, take the factor covariance itself
$\Sig = \operatorname{diag}(d) + U\,\Delta\,U\trans$ as the risk model, estimate $\mu$
by the sample mean, and trace the long-only, fully invested frontier
($\mathbf{1}\trans w = 1$, $0\le w\le 1$), the instance $\mathrm{M}_c$ of
\eqref{eq:mark} in which the gross-exposure constraint is vacuous by
Proposition~\ref{prop:hinge}. The Critical Line Algorithm returns the \emph{entire}
frontier as a sequence of \textbf{21 turning points} (Figure~\ref{fig:frontier});
between consecutive points the frontier is the exact affine segment
\eqref{eq:affine}, so these $21$ corner portfolios describe the whole efficient set
with no discretisation, the portfolio counterpart of the LASSO breakpoints of
Figure~\ref{fig:both}. The dense matrix and the structured (Woodbury)
\texttt{FactorCovariance} backend represent the \emph{same} $\Sig$ and return the
identical $21$-point frontier: by the operator reading of $H$
(Section~\ref{sec:scheme}) the structured solve changes only the cost of each step,
never the result.

\section{Shared scaffolding, particular features}
\label{sec:taxonomy}

With the identity in hand, the family is organised not by field of origin but by a
short list of features, against a large shared core. The core is all of
Section~\ref{sec:scheme}: the parametric QP, the affine segment, the homotopy, the
simplex-inherited anti-cycling. Four features distinguish the members.

\emph{Imposed versus induced active sets.} Constraints (the CLA, constrained
regression) impose faces; penalties (the LASSO, generalized lasso, SVM path) induce
them through a subgradient. Theorem~\ref{thm:identity} and
Proposition~\ref{prop:hinge} show the two are interconvertible, with the
gross-exposure ball the hinge, but their computational ergonomics differ, which is
why each field built its own version.

\emph{Monotone versus non-monotone active sets.} Forward selection and forward LARS
only \emph{add} variables; the LASSO modification and the CLA throughout both add and
drop, since coefficients can return to zero and weights can leave and re-enter their
bounds as $\lambda$ moves. The CLA corresponds to the full LASSO homotopy, not to
forward LARS.

\emph{The structure of $H$.} The scheme touches the data only through the quadratic
form: a covariance in finance, a Gram matrix in regression, a kernel matrix in the SVM
path. What $H$ is fixes both interpretation and cost, and, crucially, the homotopy
never needs it explicitly (Section~\ref{sec:comp}).

\emph{Degeneracy.} Ties and near-rank-deficiency are the recurring practical hazard;
the inherited Bland rule guarantees determinism but, as in the simplex method, does not
fully tame the adversarial worst case (Section~\ref{sec:length}).

\section{A tour of the instances}
\label{sec:tour}

Read through the taxonomy of Section~\ref{sec:taxonomy}, each member is the common
scheme with its quadratic form and face-mechanism filled in, and the differences are
exactly the four features.

\paragraph{The LASSO and least angle regression} $H=X\trans X$, $d=X\trans y$; the
active set is induced by the $\ell_1$ subgradient and is non-monotone. A coordinate
enters when its correlation with the residual reaches $\lambda$ and leaves when its
coefficient crosses zero; forward LARS is the entering-only ancestor, and the LASSO
modification restores the leave move \citep{efron2004}. This is the instance
Theorem~\ref{thm:identity} pairs with the CLA.

\paragraph{The support-vector-machine path} \citet{hastie2004svm} trace the SVM
solution as the cost parameter varies; here $H$ is a \emph{kernel} Gram matrix
$\kappa(X,X)$, the active set is the set of margin support vectors, and the events are
points crossing the margin. It is induced and non-monotone, and it is the sharpest
illustration of the operator reading of $H$: the path touches the kernel only through
the three actions of Section~\ref{sec:scheme}, never forming it densely.

\paragraph{The elastic net and the generalized lasso} The elastic net
\citep{zou2005} adds a ridge, sending $H\to X\trans X + \eta I$ and otherwise leaving
the scheme untouched: the regularisation merely conditions the quadratic form. The
generalized lasso \citep{tibshirani2011}, with penalty $\lambda\|D\beta\|_1$ for a
structured difference operator $D$ (the fused lasso, trend filtering, and graph
penalties as special cases), traces a piecewise-linear \emph{dual} path by the same
homotopy; the face-mechanism acts on $D\beta$ rather than $\beta$, a change of
coordinates, not of scheme.

\paragraph{The Critical Line Algorithm} $H=\Sig$, $d=\mu$; the active set is
\emph{imposed} by the box bounds and is non-monotone, with assets leaving and
re-entering the free set as $\lambda$ decreases, tracing the corner portfolios of
Figure~\ref{fig:frontier}. It is the oldest instance by four decades
(Section~\ref{sec:history}) and, through Theorem~\ref{thm:identity}, the portfolio
face of the LASSO.

\paragraph{Explicit model predictive control} The constrained linear-quadratic
regulator solved as a function of the state \citep{bemporad2002, tondel2003} is the
multidimensional cousin: the parameter is a \emph{vector} (the state), so the scheme's
interval of breakpoints becomes a polyhedral partition into critical regions and the
homotopy becomes region traversal. Only its one-parameter slice is literally
\eqref{eq:pqp}; explicit MPC is therefore a \emph{generalisation} of the common
scheme, not another instance of the scalar identity of Section~\ref{sec:identity},
which concerns that slice and not the full vector partition.

None of these needs machinery beyond Section~\ref{sec:scheme}; what looks like five
literatures is one algorithm with five fillings.

\section{History}
\label{sec:history}

\subsection{The Critical Line Algorithm}
\label{sec:cla-history}

The active-set treatment of the parametric quadratic program is a creation of the
1950s, and the Critical Line Algorithm is its earliest fully worked instance.
\citet{markowitz1952} posed mean--variance selection; recognising that the efficient
set is obtained by solving a quadratic program \emph{parametrically} in the
risk--return trade-off, \citet{markowitz1956} gave the Critical Line Algorithm,
announced in its title as ``the optimization of a quadratic function subject to linear
constraints,'' and developed it in full in the 1959 monograph \citep{markowitz1959}.
The method is exactly the homotopy of Section~\ref{sec:scheme}: trace the critical
line, changing the status of one asset at each turning point. It is contemporary with,
and independent of, the general active-set quadratic-programming theory of the same
years, notably \citet{wolfe1959}, who in \emph{Econometrica} adapted the simplex
method to quadratic programming and fixed the ratio-test-and-pivot lineage the homotopy
still rides.

The wider apparatus was being built in the same handful of years, and strikingly often
in the same journal. \citet{frankwolfe1956} and \citet{beale1959} gave general
quadratic-programming algorithms, while \citet{gasssaaty1955} introduced
\emph{parametric} linear programming, following a solution as a cost coefficient
varies, the linear ancestor of the homotopy used here; the Markowitz, Frank--Wolfe,
Beale, and Gass--Saaty papers all appeared in the \emph{Naval Research Logistics
Quarterly} between 1955 and 1959. By the close of the decade the parametric active-set
treatment of quadratic programs existed, in finance and in optimization alike, in
essentially the form we use today.

Two threads then ran for half a century. On the modelling side, \citet{sharpe1963}, at
Markowitz's suggestion, introduced the single-index model, writing the covariance as a
diagonal matrix plus a single rank-one term to tame the estimation and computational
burden of a dense $\Sig$. Modern multi-factor risk models generalise this
diagonal-plus-low-rank form, and it is precisely the structure the operator reading of
$H$ (Sections~\ref{sec:scheme} and~\ref{sec:comp}) exploits: the covariance the
classical CLA treats as a dense matrix is, in practice, almost never dense. On the
algorithmic side, the CLA was re-derived, refined, and scaled across decades:
\citet{perold1984} for large-scale portfolio optimization, then \citet{stein2008} and
\citet{hirschberger2010} for active-set and large-universe variants,
\citet{niedermayer2010} for a careful fast implementation, \citet{markowitz2000} for
the canonical textbook description and reference code, and \citet{bailey2013} for a
widely used open-source version; \citet{markowitz2021} gives a recent step-by-step
account extending the method to mean--semivariance. Throughout, the algorithm was
presented as a portfolio method, and its membership in a wider homotopy family was not
the question being asked.

\subsection{And how statistics came to it independently}
\label{sec:stats-history}

Statistics arrived at the same device around forty years later, by a different door,
and at first without reference to any of this. \citet{osborne2000} gave a homotopy for
the LASSO of \citet{tibshirani1996}; \citet{efron2004} gave least angle regression and
made the piecewise-linear path famous; \citet{hastie2004svm} extended it to the support
vector machine, and \citet{rosset2007} characterised, within statistics, exactly when
such a path is piecewise linear, recovering for the field the criterion the
parametric-programming literature had long had. The methods were presented as native to
regularised estimation.

The bridge back was late and partial. \citet{brodie2009} observed that a
gross-exposure-constrained Markowitz problem \emph{is} a LASSO, the special case of
Theorem~\ref{thm:identity} without general constraints. Even so, the observation
travelled as a curiosity rather than as the statement that two large algorithmic
literatures describe one method. That is the gap this paper closes: not a new
algorithm, but the recognition, made exact and general, that the algorithm was never
four.

\section{Computation: one engine, many problems}
\label{sec:comp}

The unification has a software corollary. Because every step of the homotopy accesses
$H$ only through a matrix--vector product, a solve against the active block, and an
active-to-inactive cross product, one implementation serves every instance: told the
quadratic form (as an operator) and what an event means, it traces the path. The same
loop that follows the efficient frontier follows the LASSO path when handed
$X\trans X$ and $X\trans y$. The \texttt{cvxcla} package \citep{cvxcla} realises this,
and traced the homotopy shown in Figure~\ref{fig:both}. A few deliberately light checks
confirm that
Theorem~\ref{thm:identity} holds numerically and not merely on paper. On a standard
design, the traced path agrees with \texttt{scikit-learn}'s \texttt{lars\_path} to
machine precision, about $10^{-15}$ in the coefficients, over the same breakpoints;
under a non-negativity and an inequality constraint, the constrained path matches an
independent active-set quadratic-program solver, run separately at each breakpoint,
again to machine precision (about $5\times10^{-16}$). A conic interior-point solver
agrees only to its own $\approx 10^{-8}$ floor, which is why the agreement is reported
against a direct active-set reference. On real data, the diabetes study of
\citet{efron2004} itself, the homotopy reproduces \texttt{lars\_path} to
$\approx 2\times10^{-10}$ over its thirteen breakpoints. The high-dimensional regime is
no obstacle: on a $p\gg n$ design, where $X\trans X$ is singular, the homotopy still
traces the path, the active support never exceeding $\operatorname{rank}(X)$ so that the
free block $X_\F\trans X_\F$ stays positive definite (the standing assumption of
Section~\ref{sec:scheme}), and it again matches \texttt{lars\_path}, to
$\approx 10^{-13}$ on designs with $p$ several times $n$.

\paragraph{Tracing cleanly at scale} The homotopy returns the whole long-only frontier
as an exactly ordered sequence of turning points well beyond the small illustration of
Figure~\ref{fig:frontier}. One numerical point earns its keep here. The breakpoints of a
large path crowd together, their spacing shrinking with the universe size, so ordering
them, and the simplex-style ratio test that does it, must use a tolerance \emph{relative}
to the scale of $\lambda$; a fixed absolute tolerance eventually merges distinct
breakpoints, mis-ranks them, and lets the parameter step backwards. With the relative
test, \texttt{cvxcla} returns strictly monotone frontiers, turning points in the correct
order to floating-point precision, on factor-model universes from $N=100$ up to
$N=6400$ assets, the largest agreeing turn for turn with a tight-tolerance reference; the
count of corners stays near-linear, about $N$ for an $N$-asset frontier. This is benign,
general-position behaviour; the adversarial worst case is a separate matter, taken up in
Section~\ref{sec:length}.

The exactness is not a numerical nicety: gridding the penalty and interpolating between
grid points cuts the very corners, the breakpoints, that carry the model-selection
information. Coordinate-descent and proximal solvers run on a $\lambda$-grid approximate
the path in exactly this way, whereas the homotopy returns the corners themselves.

Treating $H$ as an operator is also where the scheme meets scale. A structured
quadratic form, a diagonal-plus-low-rank factor covariance in finance or a kernel in
learning, supplies the same operations far more cheaply than a dense $H$, with no
change to the homotopy. This is not only asymptotics. Tracing the entire long-only
frontier of a $k=10$ factor universe, the diagonal-plus-low-rank
\texttt{FactorCovariance} operator returns the \emph{identical} frontier as the dense
$\Sig$ but does so about $4\times$ faster at $N=250$ assets, $11\times$ at $N=1000$,
and $36\times$ at $N=2000$ (where the dense solve takes some twenty seconds and the
operator under one); the gap widens with $N$ because each step costs $O(nk)$ rather
than $O(n^2)$ (hardware-dependent figures, reproduced by the companion script). The
portfolio analogue of a structured or kernelised design is a factor risk model, and it
is how Markowitz's algorithm reaches universe sizes its original dense formulation
could not.

\section{How long is the path?}
\label{sec:length}

The unified view sharpens the family's central open question. In practice the number
of breakpoints grows linearly in the problem size for both the LASSO and the CLA, and
this near-linearity is the homotopy's reason for being: tracing the whole path costs
about what a handful of fits would. The worst case is not benign.
\citet{mairal2012} construct LASSO instances whose path has a number of breakpoints
\emph{exponential} in the dimension, the statistical mirror of the long-known
exponential bounds on the number of critical regions of a multi-parametric QP
\citep{tondel2003}. So the same algorithm is, depending on the instance, either a
near-linear method or an exponential one, and the unified view says this dichotomy is a
property of parametric QPs as such, not of any one field's version.

Three questions follow and are, to our knowledge, open in the shared form. What data
conditions separate the benign average from the malignant worst case, and is the
near-linear behavior generic under, say, a continuous design distribution? When does
tracing the path beat solving at a single parameter with coordinate descent or a
general QP solver, as a function of the conditioning and structure of $H$? And can the
Bland tie-break, which guarantees termination but not graceful behavior on the most
degenerate large instances, be replaced by a rule with a worst-case guarantee, or shown
unnecessary for generic data? A resolution would apply to every instance at once,
which is the practical dividend of seeing them as one.

The constrained identity raises questions of its own, and the first can now be largely
answered. Proposition~\ref{prop:reg} writes the slope as
$c'(\lambda) = -\,\|(Z\trans H_{\F\F}Z)^{-1/2}Z\trans s_\Aset\|_2^2$, which vanishes
exactly when $Z\trans s_\Aset = 0$, that is, when the sign vector $s_\Aset$ lies in the
span of the active constraint normals. For a continuous design those normals span a
fixed proper subspace, and a $\pm1$ vector landing in it is a measure-zero coincidence,
so flat segments are \emph{non-generic}: away from them the penalty and constraint forms
are mutually reparametrisable. They are not impossible, and the boundary is sharp by
construction: a single equality whose normal is aligned with the support's signs,
$a = s_\Aset$, makes $s_\Aset$ an active normal, forces $Z\trans s_\Aset = 0$, and
produces a flat segment on which a whole range of penalties $\lambda$ collapses to one
budget $c$ and the reparametrisation fails. What stays open is which constraint systems
force such alignment on a set of positive measure rather than at isolated points. The worst-case breakpoint count under general
$G\beta\le h$ is also open: each inequality row adds a face that can be crossed, so the
constrained path is at least as long as the unconstrained one, but whether $p$
inequality rows can \emph{multiply} the count rather than merely add to it is not known.
And the uniqueness behind Theorem~\ref{thm:identity} rests on general position plus a
constraint qualification; the boundary where the qualification fails is the constrained
analogue of the unconstrained ties the Bland rule manages, and invites the same
analysis. These are not portfolio questions or regression questions but questions about
the shared object, which is the point.

\section{Conclusion}

Statistics, optimization, and finance each found that following the optimiser of a
parametrised convex quadratic produces a piecewise-linear path that an active set
suffices to trace, and each named the discovery after its own problem. For a scalar
parameter they are one device; explicit model predictive control, sweeping a vector
state, is the multi-parameter generalisation of that device rather than another copy
of the scalar path, and the exact identity that follows is a statement about the
scalar members. The portfolio frontier and the LASSO path are not analogous but
identical, by
a one-line substitution that survives general constraints
(Theorem~\ref{thm:identity}); the apparent variety of the wider family reduces to a
short list of features (Section~\ref{sec:taxonomy}); and one implementation, accessing
its quadratic form as an operator, traces them all. Recognising the device lets a
result, an implementation, or a worst-case analysis built for one instance serve the
rest, and turns four parallel literatures on path-following into one.
